# 地物识别功能实施计划

## 背景

基于已训练完成的 DeepLabV3+ 模型（识别6类地物：背景、水系、林地、道路、种植土地、建（构）筑物），
在现有 FastAPI 后端中新增推理接口，前端 OpenLayers 地图上以半透明色块叠加展示识别结果。

---

## 整体数据流

```
前端点击"识别"按钮
  ↓ POST /api/segment { image_id, bbox, width, height, srs }
后端从数据库取 wms_url + layer_name
  ↓ 向 GeoServer 发 WMS GetMap 请求（只拉当前视口区域）
  ↓ DeepLab 推理 → 生成半透明分割色块 PNG
  ↓ 返回 { image: "data:image/png;base64,...", bbox }
前端用 ol/source/ImageStatic 创建 ImageLayer
  ↓ 叠加在原 DOM 影像上（bbox 精确对齐，不错位）
```

## 实施步骤

### Step 1：整合 DeepLab 代码到后端项目

**操作：** 将以下文件/目录从 `E:\change_detection\deeplabv3-plus-pytorch-main` 复制到
`E:\change_detection\change_detection_backend`

| 来源 | 目标位置 | 说明 |
|------|---------|------|
| `nets/` | `change_detection_backend/nets/` | 模型网络定义 |
| `deeplab.py` | `change_detection_backend/deeplab.py` | DeeplabV3 推理类 |
| `model_data/` | `change_detection_backend/model_data/` | backbone 预训练权重 |
| `logs/logs_xception2024/best_epoch_weights.pth` | `change_detection_backend/weights/best_epoch_weights.pth` | 训练好的模型权重 |

**注意：** `deeplab.py` 中的 `model_path` 默认值需要修改为新路径 `weights/best_epoch_weights.pth`

---

### Step 2：新建 `utils/deeplab_service.py`

封装 `DeeplabV3` 为单例，FastAPI 启动时加载一次模型，避免每次请求重复加载。

**关键逻辑：**
- 提供 `load_model()` 在 lifespan 中调用
- 提供 `get_model()` 在接口中获取已加载的实例
- 推理结果返回半透明 RGBA PNG（bytes）

---

### Step 3：新建 `router/segment.py`

新增接口 `POST /api/segment`

**请求体：**
```json
{
  "image_id": 1,
  "bbox": "116.3,39.8,116.5,40.0",
  "width": 800,
  "height": 600,
  "srs": "EPSG:4326"
}
```

**处理流程：**
1. 从数据库查询 `image_id` 对应的 `wms_url`、`layer_name`（鉴权：只能查自己的影像）
2. 拼接 GeoServer WMS GetMap 请求，拉取当前视口 PNG
3. 用 `asyncio.to_thread` 在线程池中运行 DeepLab 推理（避免阻塞事件循环）
4. 生成半透明 RGBA 分割色块 PNG（背景类设为全透明）
5. 返回 base64 编码结果

**响应体：**
```json
{
  "code": 200,
  "message": "识别成功",
  "data": {
    "image": "data:image/png;base64,...",
    "bbox": "116.3,39.8,116.5,40.0"
  }
}
```

---

### Step 4：修改 `main.py`

1. lifespan 中加载 DeepLab 模型（启动时执行一次）
2. 注册 `segment.router`

---

### Step 5：前端开发

**新增功能点：**

1. **识别按钮**：在识别页地图工具栏添加"开始识别"按钮

2. **调用接口**：
   - 取当前地图视口 `extent` 和地图像素尺寸 `size`
   - 组装请求体发送 `POST /api/segment`

3. **叠加结果**：
   - 收到 base64 PNG 后，用 `ol/source/ImageStatic` 创建 `ImageLayer`
   - `imageExtent` 设为请求时的 `bbox`（精确对齐）
   - 叠加在 DOM 影像图层之上

4. **UI 交互**：
   - 识别中显示 loading 状态
   - 支持透明度滑块调节结果图层
   - "清除结果"按钮移除叠加层

## 暂不做的事

- 识别结果不入库（`detection_tasks` 表暂不写入），后续有历史记录需求再加
- 结果不发布到 GeoServer（base64 直接返回，响应更快）
- 不支持整幅大图推理（只推理当前视口区域，避免内存溢出）

## 依赖要求

后端 conda 环境 `deeplabv3` 中已安装：
- `torch 2.12.0.dev+cu128`（支持 RTX 5060 Ti）
- `torchvision`
- `Pillow`
- `requests`（调 GeoServer WMS）

FastAPI 启动需使用 `deeplabv3` 环境运行。
